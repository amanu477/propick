## Option 1: Give It ALL AT ONCE (One Shot)

```markdown
# PROJECT: Affiliate Marketing Website

## OVERVIEW
Build a complete affiliate marketing website using **Next.js 14 (App Router)**, 
**Tailwind CSS**, and **TypeScript**. The site has 3 core systems:

1. **Comparison Pages** — "Best X in 2025" and "X vs Y" pages with 
   product data, ratings, pros/cons, feature comparisons, and CTA buttons
2. **Smart Link Redirect System** — Short URLs like `/go/product-name` 
   that track clicks and redirect to affiliate URLs
3. **Link-in-Bio Page** — A beautiful mobile-first page listing all 
   recommended products organized by category, shareable on social media

---

## TECH STACK
- **Framework:** Next.js 14 with App Router
- **Styling:** Tailwind CSS
- **Language:** TypeScript
- **Database:** JSON file storage (for click tracking — easy to upgrade later)
- **Deployment:** Vercel (free tier)
- **No external backend needed**

## TYPE DEFINITIONS

```typescript
// types/index.ts

export interface Product {
  id: string
  name: string
  logo: string
  rating: number                    // 1-5 scale
  price: number                     // Monthly price
  originalPrice: number             // Before discount
  discount: string                  // e.g. "67%"
  currency: string
  billing: string                   // e.g. "monthly (2-year plan)"
  affiliateSlug: string             // Used in /go/[slug]
  badge?: string                    // e.g. "🏆 Editor's Choice"
  shortDescription: string
  features: string[]
  pros: string[]
  cons: string[]
  bestFor: string
  scores: {
    speed: number                   // 0-100
    security: number                // 0-100
    value: number                   // 0-100
    ease: number                    // 0-100
  }
  detailedReview: string
  lastUpdated: string               // ISO date
  url: string                       // Official website (non-affiliate)
}

export interface AffiliateLink {
  url: string                       // Full affiliate URL
  program: string
  commission: string
  cookieDays: number
}

export interface ClickLog {
  slug: string
  timestamp: string
  date: string
  userAgent: string
  referer: string
  country: string
}

export interface Category {
  id: string
  name: string
  title: string                     // "Best VPN Services"
  description: string
  icon: string
}

export interface LinkBioCategory {
  title: string
  links: LinkBioItem[]
}

export interface LinkBioItem {
  name: string
  description: string
  slug: string                      // affiliate redirect slug
  emoji: string
  hot?: boolean
}
```

## SAMPLE DATA (Use this structure, add your own products)

Create sample data for a **VPN niche** with these 5 products:
1. NordVPN — rating 4.8, $3.99/mo, Editor's Choice
2. ExpressVPN — rating 4.7, $6.67/mo, Fastest
3. Surfshark — rating 4.6, $2.49/mo, Best Value
4. CyberGhost — rating 4.4, $2.19/mo, Most Servers
5. ProtonVPN — rating 4.5, $4.99/mo, Most Private

Each product needs: full features list (5-6), pros (4-5), cons (2-3), 
all four scores (speed, security, value, ease), a badge, 
bestFor description, and a 2-paragraph detailedReview.

For affiliate links data, use placeholder URLs like:
`https://example.com/aff/nordvpn?ref=MYID`

---

## CORE FEATURES TO BUILD

### 1. SMART REDIRECT SYSTEM (/go/[slug])
- When user visits `/go/nordvpn`:
  - Log the click to JSON database with: slug, timestamp, date, 
    user-agent, referer, country (from headers)
  - Redirect (302) to the affiliate URL from links data
  - If slug not found, redirect to homepage
- This is a **Route Handler** (route.ts), not a page
- All affiliate links across the entire site should point to 
  `/go/slug` never directly to affiliate URLs
- This gives us: single place to update links + full click tracking

### 2. "BEST X" CATEGORY PAGES (/best/[category])
Each page should have these sections in order:

a) **Hero Section**
   - Large title: "🏆 Best [Category] in 2025"
   - Subtitle: "We tested X+ services to find the best"
   - Meta info: Updated date, read time, "Expert tested" badge

b) **Quick Answer Box**
   - Blue highlighted box
   - Says: "The best [category] in 2025 is [#1 product]"
   - Includes CTA link to that product's /go/ link

c) **Comparison Table**
   - Full-width responsive table
   - Columns: Rank, Provider (logo+name+badge), Rating (stars), 
     Price (original crossed out + discounted), Best For, CTA Button
   - Row #1 should be highlighted (yellow background, left border)
   - Hover effect on rows
   - CTA buttons go to /go/[slug]

d) **Detailed Product Cards** (one per product)
   - Rank number, logo, name, badge
   - Price with discount shown
   - Star rating
   - Short description
   - Features grid (2 columns)
   - Score bars (speed, security, value, ease) with colored 
     progress bars (green >90, yellow >70, red below)
   - Pros & Cons boxes side by side (green/red)
   - CTA button centered at bottom

e) **FAQ Section**
   - Expandable/collapsible questions using <details> tags
   - At least 3 FAQs auto-generated from product data
   - Good for SEO

f) **Affiliate Disclosure**
   - Gray box at bottom disclosing affiliate relationship

g) **SEO**
   - Dynamic metadata (title, description, openGraph)
   - Schema.org JSON-LD for Review and FAQ
   - Auto-generate for each category

### 3. "X VS Y" PAGES (/vs/[slug])
URL format: `/vs/nordvpn-vs-expressvpn`

Each page should have:

a) **Title**: "[Product 1] vs [Product 2] — Which is Better in 2025?"

b) **Head-to-Head Section**
   - 3-column grid: Product1 | VS | Product2
   - Each side shows: logo, name, price, CTA button
   - Winner side gets green border highlight

c) **Comparison Table**
   - Rows: Price, Rating, Speed Score, Security Score, Value Score, 
     Ease Score, Number of Features
   - Winner of each row gets green background + ✅
   - Count total wins per product

d) **Pros & Cons** for each product side by side

e) **Winner Box**
   - Yellow highlighted box
   - Shows the winner with score and big CTA button

f) **SEO**: Dynamic metadata + schema.org

### 4. LINK-IN-BIO PAGE (/links)

a) **Design**: Dark gradient background (purple to blue to black), 
   mobile-first, max-width 480px centered

b) **Profile Section**: Avatar image, @username, short bio, 
   social media icons

c) **Featured Deal Banner**: Animated/highlighted top deal with 
   gradient background (yellow to orange), bold text, tap to claim

d) **Link Categories**: Grouped by category with headers
   - Each link is a card with: emoji, product name, short description, 
     arrow icon, optional "HOT" badge
   - Cards have glassmorphism style (bg-white/10, backdrop-blur, 
     border-white/20)
   - Hover effect: scale up slightly, background brightens

e) **Comparison Pages Section**: Links to your best/vs pages

f) **Footer**: Small affiliate disclosure + copyright

g) All links go through `/go/[slug]`

### 5. HOMEPAGE (/)

a) **Hero**: "Find the Best Tools for Your Needs"
b) **Featured Categories**: Cards linking to /best/[category] pages
c) **Top Picks**: Show #1 product from each category
d) **Latest Comparisons**: Links to /vs/ pages
e) **Newsletter Signup** (optional, just the form UI)

### 6. ANALYTICS DASHBOARD (/dashboard)

a) **Summary Cards**: Total clicks, Today's clicks, Active links, 
   Estimated revenue (clicks × average commission)
b) **Clicks by Link Table**: Sortable table showing each slug 
   with today/total clicks
c) **Simple design**, no authentication needed (just keep URL private)
d) Data comes from the JSON click database

### 7. SEO INFRASTRUCTURE

a) **sitemap.ts**: Auto-generate sitemap.xml with all pages
b) **robots.ts**: Allow all crawlers, point to sitemap
c) **Schema.org JSON-LD**: Review schema on product pages, 
   FAQ schema on FAQ sections
d) **Meta tags**: Unique title, description, OG tags per page
e) **Semantic HTML**: Proper heading hierarchy (h1>h2>h3)
f) Add `rel="nofollow noopener sponsored"` to all affiliate links

---

## DESIGN GUIDELINES

- **Clean, modern, trustworthy** design
- **Color scheme**: White backgrounds for content, blue for accents, 
  green for CTAs/prices, red for "cons", yellow for #1/winner highlights
- **Typography**: Bold headings, readable body text
- **CTA Buttons**: Green, large, with hover animation (scale + shadow)
- **Responsive**: Mobile-first, works perfectly on all devices
- **Spacing**: Generous whitespace between sections
- **Cards**: Rounded corners (xl/2xl), subtle shadows, hover effects
- **Tables**: Alternating row colors, sticky header on mobile scroll
- **Trust signals**: "Updated [date]", "Expert tested", star ratings, 
  score bars, detailed pros/cons

---

## IMPORTANT RULES

1. **NEVER** hardcode affiliate URLs in components — always use 
   `/go/[slug]` redirect links
2. **ALWAYS** add `rel="nofollow noopener sponsored"` and 
   `target="_blank"` to affiliate links
3. **ALWAYS** include affiliate disclosure on pages with affiliate links
4. Make all product data easy to edit in the data files — no product 
   info hardcoded in page components
5. Components should be **reusable** — same ComparisonTable works 
   for VPN, hosting, or any category
6. Pages should be **statically generated** where possible for 
   fast loading and SEO
7. Use **semantic HTML** for SEO (article, section, header, etc.)
8. Make the `/go/` redirect fast — minimal processing before redirect
9. Dashboard should show useful metrics to track performance
10. All images should have alt text for SEO

---

## BUILD ORDER

Build in this exact order:
1. Types + Data files (products, links, categories)
2. Utility functions (db, tracker, seo helpers)
3. Layout (navbar + footer)
4. `/go/[slug]` redirect system
5. Components (start with CTAButton, StarRating, then bigger ones)
6. `/best/[category]` page
7. `/vs/[slug]` page
8. `/links` bio page
9. Homepage
10. Dashboard
11. Sitemap + robots + final SEO

Now build the complete project with all files. Start with step 1 
and work through each step, outputting the complete code for every file.
```
